# 菜品图片同步删除功能 Spec

## Why
当前菜品管理存在图片文件泄漏问题：删除菜品或更新菜品图片时，旧的图片文件未被同步删除，导致服务器磁盘空间浪费。

## What Changes
- 删除菜品时同步删除关联的图片文件
- 编辑菜品时如果图片URL发生变化，删除旧的图片文件

## Impact
- Affected specs: 菜品管理功能
- Affected code: `server/src/routes/admin.ts`

## ADDED Requirements

### Requirement: 菜品删除时图片同步删除
系统应当在删除菜品时，同步删除该菜品关联的图片文件。

#### Scenario: 删除带有图片的菜品
- **WHEN** 管理员删除一个有关联图片的菜品
- **THEN** 系统应删除菜品数据库记录
- **AND** 系统应同步删除该菜品关联的图片文件

#### Scenario: 删除没有图片的菜品
- **WHEN** 管理员删除一个没有关联图片的菜品
- **THEN** 系统应正常删除菜品数据库记录
- **AND** 不应尝试删除任何图片文件

### Requirement: 编辑菜品时图片变更同步删除
系统应当在编辑菜品时，如果图片URL发生变化，删除旧的图片文件。

#### Scenario: 编辑菜品并更换图片
- **WHEN** 管理员编辑菜品并将图片更换为新图片
- **THEN** 系统应更新菜品数据库记录
- **AND** 系统应删除旧的图片文件

#### Scenario: 编辑菜品并移除图片
- **WHEN** 管理员编辑菜品并移除原有图片
- **THEN** 系统应更新菜品数据库记录
- **AND** 系统应删除旧的图片文件

#### Scenario: 编辑菜品但不更改图片
- **WHEN** 管理员编辑菜品但保持原有图片不变
- **THEN** 系统应更新菜品数据库记录
- **AND** 不应删除任何图片文件

#### Scenario: 编辑菜品时新图片被其他菜品使用
- **WHEN** 管理员编辑菜品更换图片，但旧图片被其他菜品使用
- **THEN** 系统应更新菜品数据库记录
- **AND** 不应删除旧图片文件（因为其他菜品仍在使用）

## 实现方案

### 文件创建

新建临时脚本：`scripts/import-menu-data.mjs`

### 脚本逻辑（核心步骤）

```javascript
// 伪代码概要
import { chromium } from 'playwright'
import { readFile } from 'fs/promises'
import { resolve } from 'path'

// 1. 启动 Playwright 浏览器
const browser = await chromium.launch()
const page = await browser.newPage()

// 2. 导航到登录页
await page.goto('http://localhost:3000/admin')

// 3. 检查是否需要登录（如果未自动跳转到登录页）
//    如果在登录页，UI 操作登录
await page.fill('input[type="text"]', 'admin')
await page.fill('input[type="password"]', 'admin123')
await page.click('button[type="submit"]')
await page.waitForURL('**/admin/**')

// 4. 提取 admin_token cookie
const cookies = await page.context().cookies()
const token = cookies.find(c => c.name === 'admin_token').value
const cookieHeader = `admin_token=${token}`

// 5. 用 Node.js fetch 调用 API（带 cookie）
//    a. 清理现有 3 个空分类
//    b. 创建 8 个新分类，记录 id 映射
//    c. 上传 103 张图片 + 创建 103 个菜品
//    d. 创建 221 个库存项

// 6. 关闭浏览器
await browser.close()
```

### 关键实现细节

1. **Cookie 提取**：通过 `page.context().cookies()` 获取 httpOnly cookie（Playwright 可以读取 httpOnly cookie，浏览器 JS 不行）

2. **图片上传**（Node.js fetch + FormData + Blob）：
   ```javascript
   const imageBuffer = await readFile(imagePath)
   const formData = new FormData()
   formData.append('image', new Blob([imageBuffer], { type: 'image/jpeg' }), filename)
   const res = await fetch('/api/admin/upload', {
     method: 'POST',
     headers: { Cookie: cookieHeader },
     body: formData
   })
   ```

3. **图片文件名到菜名的映射**：
   - 文件名格式：`01_红烧大甲鱼.jpg`
   - 解析：去掉前缀序号和扩展名 → `红烧大甲鱼`
   - 与菜单中的菜品名称匹配

4. **幂等性**：每步操作前检查是否已存在（按名称），避免重复创建。如果脚本中途失败重跑，已创建的会被跳过。

5. **错误处理**：单个菜品/原料失败不影响其他，记录失败项，最后汇总

6. **进度输出**：每完成 10 项打印一次进度

## 假设与决策

1. **现有 3 个空分类处理**：脚本会先删除（招牌、推荐、热门），因为它们没有菜品依赖，且与我们要导入的 8 个新分类语义不同
2. **价格"时价"**：保留为字符串 `"时价"`，数据库 schema 支持（TEXT 类型），前端会原样显示
3. **原料估算**：按上表规则自动估算，不追求精确，只满足"大致估算个值"的要求
4. **图片上传**：通过后端 `/api/admin/upload` 端点，图片会被 sharp 压缩为 webp（与正常 UI 上传流程一致）
5. **菜品描述**：留空（菜单 md 中无描述信息）
6. **菜品 tags/specs**：留空数组（菜单中无此信息）
7. **菜品状态**：默认 `on_sale`（在售）
8. **库存 sort_order**：按原料在清单中的顺序（1-221）

# 一键生产环境安装运行脚本

## 概述

为红灯笼食府管理系统编写两个一键脚本，用于在生产环境完成"安装依赖 → 构建产物 → 配置环境 → 启动服务"全流程：

- `install.bat`：Windows 生产服务器使用
- `install.sh`：Linux 生产服务器使用

## 当前状态分析

### 项目运行机制（基于代码探索）

- **技术栈**：Vue 3 + Vite（前端） / Node.js + Express + sql.js（后端）
- **生产入口**：[server/src/index.ts](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/server/src/index.ts#L148-L154) — 当 `NODE_ENV=production` 时，Express 同时托管 `dist/` 下的前端静态文件和 `/api` 后端接口，统一监听 `PORT`（默认 3000）
- **构建命令**：
  - 前端：`npm run build`（vue-tsc + vite build，产物到 `dist/`）
  - 后端：`npm run build:server`（tsc，产物到 `server/dist/`）
  - 一键构建：`npm run build:production`（调用 [scripts/build-production.mjs](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/scripts/build-production.mjs)，但内部用 `xcopy`，Windows 专用）
- **进程管理**：已有 [ecosystem.config.cjs](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/ecosystem.config.cjs)，PM2 配置应用名 `red-lantern-restaurant`，入口 `server/dist/index.js`，`env_production` 设置 `NODE_ENV=production`、`PORT=3000`
- **Nginx 配置**：已有 [nginx.conf](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/nginx.conf) 示例，反代到 `127.0.0.1:3000`，`server_name` 需用户修改

### 环境变量（基于代码扫描）

代码中实际读取的环境变量：

| 变量 | 用途 | 默认行为 |
|------|------|----------|
| `PORT` | 服务端口 | 默认 3000 |
| `NODE_ENV` | 环境标识 | 非生产时走开发逻辑 |
| `JWT_SECRET` | JWT 签名密钥 | 未设置时动态生成，重启后所有 token 失效（[jwt.ts](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/server/src/utils/jwt.ts#L4-L8) 有警告） |
| `FRONTEND_URL` | 生产环境 CORS 白名单 | 未设置则 CORS origin 为 undefined |

现有 [.env](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/.env) 仅含 `PORT=3000`，缺少 `JWT_SECRET`，生产环境会导致重启后管理员/顾客全部掉线。

### 用户决策（已确认）

1. 进程管理：**PM2**（脚本自动检测并全局安装）
2. Nginx：**自动配置**（bash 版复制配置并重载；bat 版仅打印提示，因 Windows 通常不用 Nginx）
3. .env：**自动生成默认配置**（含随机 JWT_SECRET）

## 实施方案

### 文件清单

| 文件 | 类型 | 说明 |
|------|------|------|
| `install.bat` | 新增 | Windows 一键安装运行脚本 |
| `install.sh` | 新增 | Linux 一键安装运行脚本 |

### install.bat 设计（Windows）

**放置位置**：项目根目录 `c:\Users\赵晨旭\Desktop\红灯笼食府管理系统\install.bat`

**执行流程**：

```
1. 设置 UTF-8 编码（chcp 65001），启用 delayed expansion
2. 打印横幅 + 确认提示（按任意键继续）
3. 环境检查
   - where node → 未安装则报错退出，提示安装 Node.js 18+
   - where npm → 未安装则报错退出
   - node -v 输出版本，低于 18 则警告（通过解析主版本号）
4. 安装依赖
   - npm install（生产依赖 + 开发依赖，因为构建需要 vue-tsc/tsx）
5. 构建产物
   - npm run build（前端）
   - npm run build:server（后端）
   - 构建失败则退出
6. 配置 .env
   - 若 .env 不存在：生成默认 .env，包含：
     PORT=3000
     NODE_ENV=production
     JWT_SECRET=<用 PowerShell 生成 96 字符随机 hex>
     FRONTEND_URL=http://localhost:3000
   - 若 .env 已存在：检查是否含 JWT_SECRET，无则追加一行（避免覆盖用户配置）
7. 安装 PM2
   - where pm2 → 未安装则 npm install -g pm2
8. 启动服务
   - pm2 delete red-lantern-restaurant（忽略错误，清理旧进程）
   - pm2 start ecosystem.config.cjs --env production
   - pm2 save
9. 打印结果
   - 访问地址 http://localhost:3000
   - 默认管理员账号 admin / admin123
   - pm2 常用命令（pm2 status / pm2 logs / pm2 stop）
   - 提示：Windows 开机自启需执行 `pm2 start` 后运行 `pm2 save`，并安装 `pm2-windows-startup`
```

**关键实现细节**：
- 使用 `chcp 65001` 确保中文正常显示
- 错误处理：每步用 `if errorlevel 1` 检查，失败则 `exit /b 1`
- JWT_SECRET 生成：调用 `powershell -Command "[guid]::NewGuid().ToString('N') + [guid]::NewGuid().ToString('N')"` 生成 64 字符随机串
- 不使用 `&&` 连接命令（遵循用户规则）

### install.sh 设计（Linux）

**放置位置**：项目根目录 `c:\Users\赵晨旭\Desktop\红灯笼食府管理系统\install.sh`

**执行流程**：

```
1. set -euo pipefail，切换到脚本所在目录
2. 检查 root 权限（Nginx 配置需要），非 root 则提示并退出
3. 打印横幅
4. 环境检查
   - command -v node || 报错退出，提示安装 Node.js 18+
   - command -v npm || 报错退出
   - 解析 node 版本，低于 18 则警告退出
5. 安装依赖
   - npm install
6. 构建产物
   - npm run build
   - npm run build:server
7. 配置 .env
   - 若 .env 不存在：生成默认 .env，JWT_SECRET 用 openssl rand -hex 48 生成
   - 若已存在且无 JWT_SECRET：追加
8. 安装 PM2
   - command -v pm2 || npm install -g pm2
9. 启动服务
   - pm2 delete red-lantern-restaurant 2>/dev/null || true
   - pm2 start ecosystem.config.cjs --env production
   - pm2 save
   - pm2 startup systemd（自动配置开机自启，执行其输出）
10. 配置 Nginx
    - 检查 nginx 是否安装：command -v nginx || apt-get install -y nginx（仅 Debian/Ubuntu）
    - 复制 nginx.conf 到 /etc/nginx/sites-available/red-lantern.conf
    - 创建软链接 /etc/nginx/sites-enabled/red-lantern.conf
    - nginx -t 测试配置
    - systemctl reload nginx（或 nginx -s reload）
    - 提示用户修改 server_name（sed 替换 your-domain.com 为实际域名，或提示手动修改）
11. 打印结果
    - 访问地址
    - 默认账号
    - pm2 常用命令
    - Nginx 配置文件位置
    - 防火墙提示（ufw allow 80,443）
```

**关键实现细节**：
- `#!/bin/bash` + `set -euo pipefail`
- 必须以 root 运行（因 Nginx 配置）
- JWT_SECRET：`openssl rand -hex 48`
- Nginx 配置：直接复制项目的 nginx.conf，提示用户修改 `server_name`
- 兼容 Debian/Ubuntu（apt-get），其他发行版提示手动安装 nginx
- 切换工作目录：`cd "$(dirname "$0")"`

### 共同设计原则

1. **幂等性**：脚本可重复执行，不会破坏已有配置
2. **不覆盖用户配置**：.env 已存在则只追加缺失项，不重写
3. **清晰的错误提示**：每步失败都打印明确原因和解决建议
4. **彩色输出**（bash）/ 分隔线（bat）：提升可读性
5. **不修改 ecosystem.config.cjs**：直接使用现有 PM2 配置
6. **不修改 nginx.conf 源文件**：复制到系统目录

## 假设与决策

### 假设

1. 用户已在生产服务器准备好 Node.js 18+ 环境（脚本检查但不自动安装 Node.js，因系统包管理器差异大）
2. Linux 服务器为 Debian/Ubuntu 系（apt-get），其他发行版脚本会提示手动安装 Nginx
3. PM2 全局安装路径在 PATH 中（npm install -g 后通常自动生效）
4. 项目代码已通过 git clone 或其他方式部署到服务器，脚本在项目根目录运行

### 决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 脚本位置 | 项目根目录 | 用户 clone 后直接运行，最直观 |
| JWT_SECRET 生成 | 脚本自动生成随机值 | 避免用户忘记设置导致重启掉线 |
| .env 策略 | 不存在则创建，存在则补全缺失项 | 幂等且不破坏用户配置 |
| Nginx（Windows） | 仅打印提示 | Windows 生产环境通常不用 Nginx，用 IIS 或直接暴露 Node |
| Nginx（Linux） | 自动复制配置 + 重载 | 用户要求自动配置 |
| PM2 开机自启 | bash 版自动配置，bat 版提示手动 | Linux systemd 标准做法，Windows 需额外工具 |
| 构建命令 | 分别执行 build 和 build:server | 不用 build:production（含 Windows 专用的 xcopy） |

## 验证步骤

### Windows 验证（install.bat）

1. 在 Windows 环境双击或命令行运行 `install.bat`
2. 检查：
   - 控制台无报错，最后打印访问地址
   - `dist/` 目录生成，含 `index.html`
   - `server/dist/` 目录生成，含 `index.js`
   - `.env` 文件含 `JWT_SECRET`
   - `pm2 status` 显示 `red-lantern-restaurant` 为 online
   - 浏览器访问 `http://localhost:3000` 显示首页
   - `http://localhost:3000/health` 返回 `{"status":"ok",...}`

### Linux 验证（install.sh）

1. 在 Linux 服务器执行 `sudo bash install.sh`
2. 检查：
   - 脚本无报错完成
   - `dist/`、`server/dist/` 生成
   - `.env` 含 `JWT_SECRET`
   - `pm2 status` 显示 online
   - `nginx -t` 通过
   - `curl http://localhost:3000/health` 返回 ok
   - `curl http://localhost/health` 经 Nginx 返回 ok（需先改 server_name）

### 幂等性验证

- 再次运行脚本，应无报错，服务正常重启，.env 不被覆盖

## 风险与限制

1. **Node.js 未安装**：脚本不自动安装 Node.js，仅提示。因不同 OS 安装方式差异大（nvm/apt/choco），自动安装可能引入问题。
2. **Nginx server_name**：脚本不会自动猜测域名，需用户手动修改 `/etc/nginx/sites-available/red-lantern.conf` 中的 `server_name`。
3. **端口冲突**：若 3000 端口被占用，PM2 启动会失败，脚本会显示 PM2 错误日志。
4. **权限问题**（Linux）：非 root 运行会在 Nginx 配置步骤失败，脚本会提前检查并退出。

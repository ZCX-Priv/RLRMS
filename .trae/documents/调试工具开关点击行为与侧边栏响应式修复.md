# 调试工具开关点击行为与 toast/侧边栏不显示修复

## 摘要

修复两个问题：
1. 点击"调试工具"开关时，开关视觉上会先打开再弹窗——应保持关闭状态，确认后开关打开与 toast 弹出**同时进行**。
2. 确认启用后**看不到 toast**，且侧边栏不实时出现"调试工具"菜单项——共同根因是 HMR 未完全生效，store 实例为旧结构，需重启 dev server。

## 当前状态分析

### 问题1：开关视觉闪烁的根因

[SettingsView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/admin/views/SettingsView.vue#L365-L372) 第 365-372 行开关模板：

```vue
<label class="toggle-switch">
  <input
    type="checkbox"
    :checked="appStore.devMode"
    @change="handleDevModeToggle"
  />
  <span class="toggle-slider"></span>
</label>
```

当前用 `@change` 事件。**问题机制**：
1. 用户点击 checkbox → 浏览器**立即**将 `input.checked` 置为 true（原生默认行为，`:checked` 绑定无法阻止）
2. `change` 事件触发 → `handleDevModeToggle` 执行 → 显示确认弹窗
3. 此时 `appStore.devMode` 仍为 false，Vue 在**下一个 tick** 才把 `input.checked` 恢复为 false
4. 用户在步骤1~3之间看到开关短暂打开的视觉闪烁

## 提议的改动

### 改动1：修复开关点击行为（SettingsView.vue）

**文件**：[src/admin/views/SettingsView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/admin/views/SettingsView.vue)

**做什么**：
1. 模板：将 input 的 `@change="handleDevModeToggle"` 改为 `@click.prevent="handleDevModeClick"`。
2. 脚本：将 `handleDevModeToggle(e: Event)` 替换为 `handleDevModeClick()`（无参数，直接读取 `appStore.devMode` 当前状态判断）。

**为什么**：
- `@click.prevent` 在 click 阶段阻止 checkbox 原生默认行为（切换 `checked`），点击后开关视觉状态**不会改变**，完全由 `:checked="appStore.devMode"` 绑定控制，消除视觉闪烁。
- label 包含 input 时，点击 label 非 input 区域会转发为 input 的 click 事件，同样被 input 上的 `@click.prevent` 拦截，行为一致。
- 处理函数直接判断 `appStore.devMode` 当前值：为 false（关闭）则弹确认框，为 true（开启）则直接关闭。不再依赖 `target.checked`（因为 prevent 后 checked 不变）。

**怎么做**：

模板（第 365-372 行）：
```vue
<label class="toggle-switch">
  <input
    type="checkbox"
    :checked="appStore.devMode"
    @click.prevent="handleDevModeClick"
  />
  <span class="toggle-slider"></span>
</label>
```

脚本（替换第 52-60 行的 `handleDevModeToggle`）：
```ts
// 开关点击处理：打开时弹出警告确认（开关保持关闭），关闭时直接生效
function handleDevModeClick() {
  if (appStore.devMode) {
    // 当前开启，点击则直接关闭
    appStore.setDevMode(false)
  } else {
    // 当前关闭，点击则弹出确认（开关保持关闭状态）
    showDevModeConfirm.value = true
  }
}
```

### 改动2：重启 dev server（解决 toast 不显示 + 侧边栏不刷新）

**不涉及代码修改**。Toast.vue、LayoutView.vue、app.ts 的代码逻辑均正确，无需改动。

**操作**：停止当前运行的 dev server，重新运行 `npm run dev`，让 app.ts 的结构性改动（`toasts` 数组、`devMode`/`setDevMode`/`loadDevMode`）在 store 实例中完整注册。

**为什么**：Pinia store 文件的结构性改动（新增 state/action、重构 toast 为数组）在 HMR 下不会重新初始化已有 store 实例，导致新成员不存在于运行时实例中。重启 dev server 让 store 从零初始化，toast 渲染与侧边栏响应式都将正常工作。

## 假设与决策

1. **用 `@click.prevent` 而非 `@change`**：`@change` 无法阻止 checkbox 原生切换 checked 的行为，必然产生视觉闪烁；`@click.prevent` 在 click 阶段阻止默认行为，是解决此类 toggle 闪烁的标准做法。
2. **不在 label 上加 @click.prevent**：label 包含 input 时，点击 label 会转发为 input 的 click 事件，input 上的 `@click.prevent` 已能覆盖所有点击场景（点 input 本身或点 label 区域）。无需在 label 上重复绑定。
3. **confirmDevMode 顺序不改**：`setDevMode` 与 `showToast` 同步调用、同时生效，已满足用户"同时进行"的要求。
4. **toast 与侧边栏代码不改**：Toast.vue 读 `appStore.toasts`、LayoutView.vue 读 `appStore.devMode` 均是正确的响应式用法，问题由 HMR 引起，重启即可，不引入不必要的改动。
5. **关闭开关仍不弹窗**：用户只要求"打开时弹窗警告"，关闭时直接生效，保持上一轮的决策。

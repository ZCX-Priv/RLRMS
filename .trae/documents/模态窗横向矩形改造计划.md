# 模态窗横向矩形改造计划

## 一、任务概述

将所有基于 `Modal.vue` 组件的模态窗从当前的"竖长方形"外观改为"横长方形"外观。通过**增加宽度**和**对表单内容采用横向布局**来实现。自定义模态窗（ConfirmDialog、ClientLoginModal、progress-modal、table-full-modal）不在本次改造范围内。

## 三、改造方案

### 3.1 第一步：修改 Modal.vue 尺寸（增加宽度）

**文件**：`src/shared/components/Modal.vue`

**修改内容**（第 80-93 行）：

```css
.modal-sm {
  width: 100%;
  max-width: 480px;  /* 320px → 480px */
}

.modal-md {
  width: 100%;
  max-width: 640px;  /* 480px → 640px */
}

.modal-lg {
  width: 100%;
  max-width: 800px;  /* 640px → 800px */
}
```

**理由**：统一增加各档尺寸宽度，使模态窗整体呈现横向矩形外观。sm 从 320 增至 480（增 50%），md 从 480 增至 640（增 33%），lg 从 640 增至 800（增 25%）。

### 3.2 第二步：表单类模态窗横向布局改造

对字段数 ≥ 3 的单列纵向表单，改为 2 列 grid 布局，参考 DishesView.vue 的 `.form-grid` 模式。

#### 3.2.1 admin/views/UsersView.vue（添加/编辑用户，5 字段）

**当前**：5 字段（用户名、密码、称呼、手机号、角色）单列纵向堆叠。

**改造**：
- 将 `.form-content` 的 `flex-direction: column` 改为 `display: grid; grid-template-columns: repeat(2, 1fr); gap: var(--spacing-md);`
- 字段排列：用户名 | 密码（第1行），称呼 | 手机号（第2行），角色（跨2列 full-width，第3行）
- 添加响应式：`@media (max-width: 640px) { grid-template-columns: 1fr; }`

#### 3.2.2 admin/views/InventoryView.vue（添加/编辑物料，4 字段）

**当前**：4 字段（物料名称、数量、单位、预警阈值）单列纵向堆叠。

**改造**：
- 将 `.form-content` 改为 2 列 grid 布局
- 字段排列：物料名称 | 单位（第1行），数量 | 预警阈值（第2行）
- 添加响应式适配

#### 3.2.3 admin/views/TablesView.vue（添加/编辑桌位，3 字段）

**当前**：3 字段（桌位编号、桌位名称、容纳人数）单列纵向堆叠。

**改造**：
- 将 `.form-content` 改为 2 列 grid 布局
- 字段排列：桌位编号 | 桌位名称（第1行），容纳人数（跨2列 full-width，第2行）
- 添加响应式适配

#### 3.2.4 admin/views/SettingsView.vue（修改密码，3 字段）

**当前**：3 字段（旧密码、新密码、确认密码）单列纵向堆叠。

**改造**：
- 将 `.form-content` 改为 2 列 grid 布局
- 字段排列：旧密码（跨2列 full-width，第1行），新密码 | 确认密码（第2行）
- **注意**：旧密码单独一行是因为它与新密码是不同概念；新密码和确认密码并排便于对照输入
- 添加响应式适配

### 3.3 第三步：不改造的模态窗说明

以下模态窗**不做横向布局改造**，仅受益于第一步的宽度增加：

| 模态窗 | 不改造原因 |
|--------|-----------|
| DishesView.vue (菜品) | 已是 2 列 grid 布局 |
| DishesView.vue (分类) | 仅 1 个字段，无需横向 |
| DishDetailView.vue (规格) | 规格按钮组+数量控制，纵向更符合交互 |
| DishCard.vue (规格) | 同上 |
| 所有信息展示类 | 内容性质决定，宽度增加已足够改善外观 |

## 五、假设与决策

1. **尺寸增幅决策**：sm 增至 480px（与原 md 一致），md 增至 640px（与原 lg 一致），lg 增至 800px。这样保持各档之间 160px 的等差递增。
2. **2 列布局决策**：统一采用 2 列 grid 而非 3 列，避免字段过密影响可读性。
3. **响应式决策**：所有 2 列 grid 在 `max-width: 640px` 时回退为单列，保证移动端可用性。
4. **full-width 使用决策**：奇数个字段时，最后一个字段或语义上需独占一行的字段使用 `grid-column: span 2`。
5. **不改自定义模态窗**：遵循用户明确要求，ConfirmDialog、ClientLoginModal、progress-modal、table-full-modal 保持原样。

1. **启动开发服务器**：`npm run dev`
2. **检查 Modal.vue 尺寸**：打开任意模态窗，确认宽度明显增加，呈现横向矩形
3. **检查表单横向布局**：
   - 后台管理 → 用户管理 → 添加/编辑用户：确认 5 字段为 2 列布局
   - 后台管理 → 物料管理 → 添加/编辑物料：确认 4 字段为 2 列布局
   - 后台管理 → 桌位管理 → 添加/编辑桌位：确认 3 字段为 2 列布局
   - 后台管理 → 设置 → 修改密码：确认 3 字段为 2 列布局
4. **检查响应式**：将浏览器窗口缩小到 640px 以下，确认 2 列布局回退为单列
5. **检查未改造模态窗**：确认信息展示类模态窗宽度增加但布局正常，无内容溢出
6. **运行类型检查**：`npm run typecheck`（如有）
7. **运行 lint**：`npm run lint`（如有）

# 调试工具开关确认弹窗与实时响应

## 摘要

当用户在"系统设置"页面打开"调试工具"开关时，先弹出警告确认对话框（确认/取消）。用户确认后，显示 success 状态的 toast"已启用调试工具"，且侧边栏立即出现"调试工具"菜单项，无需刷新页面。

## 提议的改动

### 改动 1：将 `devMode` 提升到 `useAppStore`

**文件**：[src/stores/app.ts](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/stores/app.ts)

**做什么**：
- 新增 `devMode` ref（默认 `false`）。
- 新增 `loadDevMode()` 函数：从 IndexedDB 读取 `'devMode'` 并设置 `devMode.value`。
- 新增 `setDevMode(val: boolean)` 函数：设置 `devMode.value = val` 并持久化到 IndexedDB。
- 在 return 中导出 `devMode`、`loadDevMode`、`setDevMode`。

**为什么**：Pinia store 的状态是响应式的，多个组件共享同一份状态。`SettingsView` 调用 `setDevMode(true)` 后，`LayoutView` 中读取 `appStore.devMode` 的 computed 会立即重新计算，侧边栏立即出现"调试工具"菜单项。

**怎么做**（关键代码）：
```ts
const devMode = ref(false)

async function loadDevMode() {
  const saved = await getItem<boolean>('devMode')
  if (saved) devMode.value = true
}

async function setDevMode(val: boolean) {
  devMode.value = val
  await setItem('devMode', val)
}
```
在 return 中追加：`devMode, loadDevMode, setDevMode`。

### 改动 2：改造 `SettingsView.vue` 的开关逻辑

**文件**：[src/admin/views/SettingsView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/admin/views/SettingsView.vue)

**做什么**：
1. 移除本地 `const devMode = ref(false)`（第 43 行）。
2. 移除 `onMounted` 中读取 devMode 的代码（第 48-50 行），改为调用 `appStore.loadDevMode()`。
3. 移除 `watch(devMode, ...)` 持久化代码（第 53-56 行）——持久化改由 store 负责。
4. 新增 `const showDevModeConfirm = ref(false)` 控制确认弹窗。
5. 新增 `handleDevModeToggle(e: Event)` 函数处理开关切换：
   - 若用户切换到"开"（`target.checked === true`）：显示确认弹窗，不修改 store 状态（由于 `:checked` 绑定 `appStore.devMode` 仍为 false，Vue 会在 nextTick 把 DOM 恢复为未勾选状态）。
   - 若用户切换到"关"（`target.checked === false`）：直接调用 `appStore.setDevMode(false)`。
6. 新增 `confirmDevMode()` 函数：确认弹窗的 confirm 回调，调用 `appStore.setDevMode(true)`，关闭弹窗，显示 `appStore.showToast('已启用调试工具', 'success')`。
7. 新增 `cancelDevMode()` 函数：确认弹窗的 cancel 回调，仅关闭弹窗。
8. 引入 `ConfirmDialog` 组件（使用 `defineAsyncComponent`，与项目其他视图一致）。
9. 修改模板：
   - 开关 input：将 `v-model="devMode"` 改为 `:checked="appStore.devMode" @change="handleDevModeToggle"`。
   - 在模板末尾追加 `<ConfirmDialog>` 组件，`type="warning"`，`title="启用调试工具"`，`message="调试工具包含高级功能，仅供开发人员使用，误操作可能影响系统数据。确定要启用吗？"`，`confirmText="启用"`，`cancelText="取消"`。

**为什么**：
- `:checked` 单向绑定 + `@change` 拦截，可以在用户打开开关时弹出确认框，而不会立即改变状态。
- 关闭开关不需要警告（用户只要求"打开时"弹窗警告）。
- 确认后调用 store 的 `setDevMode`，触发响应式更新。

**怎么做**（关键代码片段）：
```ts
// 顶部引入
const ConfirmDialog = defineAsyncComponent(() => import('@/shared/components/ConfirmDialog.vue'))

const showDevModeConfirm = ref(false)

function handleDevModeToggle(e: Event) {
  const target = e.target as HTMLInputElement
  if (target.checked) {
    // 打开：弹出警告确认
    showDevModeConfirm.value = true
  } else {
    // 关闭：直接生效
    appStore.setDevMode(false)
  }
}

function confirmDevMode() {
  showDevModeConfirm.value = false
  appStore.setDevMode(true)
  appStore.showToast('已启用调试工具', 'success')
}

function cancelDevMode() {
  showDevModeConfirm.value = false
}
```

模板（开关部分）：
```vue
<label class="toggle-switch">
  <input
    type="checkbox"
    :checked="appStore.devMode"
    @change="handleDevModeToggle"
  />
  <span class="toggle-slider"></span>
</label>
```

模板（末尾追加确认弹窗）：
```vue
<ConfirmDialog
  :show="showDevModeConfirm"
  title="启用调试工具"
  message="调试工具包含高级功能，仅供开发人员使用，误操作可能影响系统数据。确定要启用吗？"
  confirm-text="启用"
  cancel-text="取消"
  type="warning"
  @confirm="confirmDevMode"
  @cancel="cancelDevMode"
/>
```

```ts
  await appStore.loadDevMode()

const navItems = computed(() => {
  const items = [
    // ...原有项
  ]
  if (appStore.devMode) {
    items.push({ icon: Code, label: '调试工具', path: '/admin/debug' })
  }
  return items
})
```

## 假设与决策

1. **关闭开关时不弹窗**：用户明确要求"打开时弹窗警告"，关闭时直接生效，不弹窗、不 toast。侧边栏菜单项的消失本身就是反馈。
2. **确认弹窗类型用 `warning`**：用户说"警告"，ConfirmDialog 支持 `warning` 类型（黄色图标），比 `danger`（红色）更合适，因为启用调试工具不是破坏性操作。
3. **Toast 消息文案**："已启用调试工具"，与用户原话一致。
4. **不新建独立 store**：`devMode` 是全局应用状态，放在 `useAppStore` 中即可，无需新建 `useDebugStore`，避免过度设计。
5. **`loadDevMode()` 在两个组件的 onMounted 中都调用**：保证无论先进入哪个页面，store 都能加载到正确状态。重复加载无副作用（只是覆盖为相同的值）。
6. **保留 IndexedDB 持久化**：用户期望刷新后状态保持，所以仍用 IndexedDB 存储键 `'devMode'`，只是读写统一由 store 负责。

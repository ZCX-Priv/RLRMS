# 为购物车栏（未展开态）添加边框，并统一展开态边框方案

## 摘要

用户要求：购物车未展开时，`.cart-bar` 也应该显示边框（与展开态视觉一致）。当前 `.cart-bar` 基础样式只有 `box-shadow`，无边框。本计划在 `.cart-bar` 基础样式中添加 `border`，同时将展开态 `.cart-expanded + .cart-bar` 的 `outline` 方案统一为 `border` 方案，确保展开拼接处无缝、左右对齐，且未展开/展开两种状态都有连贯的边框表现。

**目的**：展开态改用 border 并去掉顶边框，与 `.cart-expanded`（底边留空）无缝拼接；左右 border 位置与 `.cart-expanded` 完全一致，消除错位与中间线条。

```css
}
```

变更点：
- 移除 `outline: 1px solid var(--color-border-light);` 与 `outline-offset: -1px;`

- 展开态下，`.cart-bar` 继承基础样式的四边 border，此处通过 `border-top: none` 去掉顶边，与 `.cart-expanded` 的底边留空处对接。

## 方案原理（两态边框表现）

**未展开态**（仅 `.cart-bar`）：
- 四边 `border: 1px solid` + `box-shadow` 浮起 + 四角 `var(--radius-xl)` 圆角。
- 视觉：带边框的浮起圆角卡片。

**展开态**（`.cart-expanded` + `.cart-bar`）：
- `.cart-expanded`：上 + 左 + 右 border，底边 none。
- `.cart-bar`：下 + 左 + 右 border，顶边 none（覆盖基础四边 border 的 top）。
- 拼接处双方均无边框 → **中间无线条**。
- 左右 border 均在 `border-box` 内侧紧贴圆角 → **无错位**。
- 整体形成完整连续的圆角矩形边框，`box-shadow: none`（展开时不再浮起）。

## 假设与决策

1. **假设**：全局 `box-sizing: border-box` 生效，添加 border 不会撑大 `.cart-bar`，布局尺寸不变。已通过 [src/style.css:117-123](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/style.css#L117-L123) 验证。
2. **决策**：未展开态保留 `box-shadow`。理由：用户仅要求"加边框"，未要求移除阴影；未展开态作为独立浮起卡片，阴影提供层次感，border 提供轮廓清晰度，两者共存视觉合理。展开态保持 `box-shadow: none`（与上方拼接为一体，不需浮起）。
3. **决策**：展开态 `.cart-expanded + .cart-bar` 统一为 border 方案。理由：当前 outline 方案与 `.cart-expanded` 的 border 方案不匹配，会导致左右错位与中间线条（即用户上一轮反馈的问题）。统一为 border 方案是修复该问题的必要连带变更，否则"未展开加边框"后展开态会因 `.cart-bar` 继承的 border-top 在拼接处产生新线条。
4. **不改动项**：`.cart-expanded` 样式保持现状（已是正确的 border 方案）；`.cart-expanded-header` 的 `border-bottom` 不变（header 与 body 的分隔线，与本次无关）。

# 桌位翻页区域高度固定 + 展开按钮样式简化

## 概述

针对用户反馈的两个 UI 问题进行修复：
1. 桌位翻页时，最后一页桌位数量少导致桌位区域高度变小
2. "展开全部"按钮样式过于花里胡哨，应改为黑色字、无背景色

## 当前状态分析

### 问题 1：桌位区高度随页内桌位数变化

**文件**：[OrderConfirmView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/client/views/OrderConfirmView.vue#L654-L658)

当前 `.table-grid` 样式：
```css
.table-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: var(--spacing-sm);
}
```

**问题原因**：grid 布局没有固定行数。当翻到最后一页只有少量桌位时（例如总共 11 个桌位，每页 9 个，第二页只有 2 个），grid 只渲染 1 行，导致整个桌位区域高度缩小，视觉上不稳定。

### 问题 2：展开按钮样式过于花哨

**文件**：
- [OrderConfirmView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/client/views/OrderConfirmView.vue#L582-L599)
- [OrderDetailView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/client/views/OrderDetailView.vue#L511-L528)

当前 `.items-toggle` 样式：
```css
.items-toggle {
  ...
  color: var(--color-primary);           /* 红色文字 */
  background-color: var(--color-bg-tertiary);  /* 浅红背景 */
  border-radius: var(--radius-md);       /* 圆角 */
  transition: background-color var(--transition-fast);
}

.items-toggle:hover {
  background-color: var(--color-border-light);  /* hover 背景变化 */
}
```

**问题原因**：按钮有红色文字、浅红背景、圆角、hover 背景变化，视觉过于突出。用户要求改为黑色字、无背景色的简洁按钮。

## 提议变更

### 变更 1：固定桌位区域高度

**文件**：`src/client/views/OrderConfirmView.vue`

**修改 `.table-grid` 样式**（第 654-658 行）：

```css
.table-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: repeat(3, minmax(72px, auto));
  gap: var(--spacing-sm);
}
```

**原理**：
- `grid-template-rows: repeat(3, minmax(72px, auto))` 强制 grid 始终保持 3 行结构
- `minmax(72px, auto)` 确保每行最小高度 72px（足够容纳桌位名称+容量两行文字+padding），有内容时按 auto 自适应展开
- 无论当前页有 1 个还是 9 个桌位，grid 都会占据 3 行的空间，保持视觉稳定

**为什么选择 72px**：
- `.table-item` 的 padding 为 `var(--spacing-md)` = 16px（上下共 32px）
- 内容：`table-name` (0.875rem ≈ 14px) + `gap` (0.25rem = 4px) + `table-capacity` (0.75rem ≈ 12px) ≈ 30px
- 单个 item 最小高度 ≈ 32px + 30px = 62px
- 取 72px 留出余量，确保不会拥挤

### 变更 2：简化展开按钮样式

**文件**：
- `src/client/views/OrderConfirmView.vue`（第 582-599 行）
- `src/client/views/OrderDetailView.vue`（第 511-528 行）

**修改 `.items-toggle` 样式**：

```css
.items-toggle {
  display: flex;
  align-items: center;
  justify-content: center;
  gap: var(--spacing-xs);
  width: 100%;
  padding: var(--spacing-sm);
  margin-top: var(--spacing-sm);
  font-size: 0.875rem;
  color: var(--color-text-primary);
  transition: opacity var(--transition-fast);
}

.items-toggle:hover {
  opacity: 0.7;
}
```

**变更点**：
- `color`: `var(--color-primary)` → `var(--color-text-primary)`（红色改为黑色/主文字色）
- 移除 `background-color: var(--color-bg-tertiary)`（无背景色）
- 移除 `border-radius: var(--radius-md)`（无背景时无需圆角）
- `transition`: `background-color` → `opacity`（改为透明度过渡）
- `:hover`: 移除背景色变化，改为 `opacity: 0.7`（轻微透明反馈）

## 假设与决策

1. **桌位区域固定 3 行**：每页最多 9 个桌位（3行×3列），因此固定 3 行高度即可覆盖所有情况。当桌位不足 9 个时，空位会留白但保持高度一致。

2. **72px 最小行高**：基于当前 `.table-item` 的 padding 和内容高度计算得出，确保桌位卡片不会拥挤。如果内容超过 72px（例如桌位名称很长），`auto` 会自动展开。

3. **按钮 hover 效果**：保留轻微的透明度变化（0.7）作为交互反馈，避免按钮看起来完全静态。如果用户希望完全无反馈，可以移除 hover 样式。

4. **同步修改两个文件**：`OrderConfirmView.vue` 和 `OrderDetailView.vue` 都有相同的 `.items-toggle` 样式，需要同步修改以保持一致。

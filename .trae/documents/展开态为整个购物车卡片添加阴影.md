# 展开态为整个购物车卡片添加阴影

## 摘要

用户要求：购物车展开后，整个购物车卡片（`.cart-expanded` + `.cart-bar` 拼接体）应有阴影。当前展开态 `.cart-expanded + .cart-bar` 设置了 `box-shadow: none`，且 `.cart-expanded` 本身无阴影，导致展开态整体无阴影浮起感。本计划通过 `:has()` 选择器在展开态给容器 `.cart-container` 添加 `filter: drop-shadow()`，沿拼接体的圆角轮廓投射单一连贯阴影，避免分别给两个子元素加 box-shadow 导致的拼接处双阴影问题。

**问题**：展开态两个子元素都无 box-shadow，整体无阴影。

**方案选型分析**（为何用 `filter: drop-shadow` 而非 `box-shadow`）：

- **给 `.cart-container` 加 box-shadow**：容器是矩形（无圆角），box-shadow 按矩形框投射，四角会露出直角阴影，而子元素是圆角拼接体 → 四角阴影与圆角不匹配，视觉错位。**不可行**。
- **分别给 `.cart-expanded` 和 `.cart-bar` 加 box-shadow**：拼接处两个 box-shadow 叠加，中间出现更深的阴影线。**不可行**。
- **只给 `.cart-expanded` 加 box-shadow**：向下投射的阴影被下方不透明的 `.cart-bar` 遮挡，下半部分外侧无阴影。**不可行**。
- **给 `.cart-container` 加 `filter: drop-shadow()`**：drop-shadow 按容器渲染结果的 **alpha 轮廓**（即子元素拼接体的圆角矩形）投射单一连贯阴影，完美匹配圆角形状，无拼接处双阴影。**最佳方案**。

**触发方式**：用 `:has(.cart-expanded)` 选择器，仅当容器内存在 `.cart-expanded`（即展开态）时启用 drop-shadow。`:has()` 在 Vue 3 scoped 模式下正常工作，且现代浏览器（Chrome 105+/Safari 15.4+/Firefox 121+）均支持，2026 年项目无兼容问题。

## 方案原理（两态阴影表现）

**未展开态**（仅 `.cart-bar`）：
- `.cart-container` 无 filter（不匹配 `:has(.cart-expanded)`）。
- `.cart-bar` 自身 `box-shadow: 0 4px 20px rgba(0,0,0,0.15)` 提供浮起阴影。
- 视觉：带边框 + 阴影的圆角浮起卡片。

**展开态**（`.cart-expanded` + `.cart-bar`）：
- `.cart-container:has(.cart-expanded)` 匹配 → 容器加 `filter: drop-shadow(0 4px 20px rgba(0,0,0,0.15))`。
- drop-shadow 沿拼接体圆角矩形轮廓投射单一连贯阴影。
- `.cart-bar` 展开态 `box-shadow: none`，`.cart-expanded` 无 box-shadow → 无双阴影叠加。
- 视觉：整个展开卡片（含上下圆角）有连贯阴影浮起。

## 假设与决策

1. **假设**：`:has()` 选择器在项目目标浏览器中可用。依据：Chrome 105+（2022.08）、Safari 15.4+（2022.03）、Firefox 121+（2023.12）均支持；项目为 2026 年现代餐厅点餐系统，无遗留旧浏览器约束。
2. **假设**：`:has()` 在 Vue 3 `<style scoped>` 中正常工作。依据：Vue 3 的 scoped 样式对 `:has()` 等现代选择器透传处理，会给相关元素加 scoped 属性，不影响 `:has()` 语义。
3. **决策**：选择 `filter: drop-shadow` 而非 `box-shadow`。理由：容器是矩形无圆角，box-shadow 会按矩形投射导致四角直角阴影与子元素圆角不匹配；drop-shadow 按 alpha 轮廓投射，完美贴合圆角拼接体。详见"方案选型分析"。
4. **决策**：阴影值复用 `0 4px 20px rgba(0, 0, 0, 0.15)`（与未展开态 `.cart-bar` box-shadow 一致）。理由：保持两态阴影观感统一，避免展开/收起切换时阴影突变。
5. **决策**：展开态不额外给 `.cart-expanded` 或 `.cart-bar` 加 box-shadow。理由：drop-shadow 已提供整体阴影，再加 box-shadow 会导致拼接处阴影叠加（双阴影线）。
6. **性能考量**：`filter: drop-shadow` 每帧按 alpha 轮廓重新计算，性能略低于 box-shadow。但本场景形状简单（圆角矩形），且仅在展开态启用，过渡动画时形状变化平滑，性能可接受。

# 解决数据保存/刷新时页面闪烁问题

## 问题分析

通过代码审查，发现以下导致闪烁的根本原因：

### 1. 数据加载时的闪烁
多个视图文件（`HomeView.vue`, `DishesView.vue`, `DashboardView.vue`, `TablesView.vue` 等）采用以下模式：
```
loading = true → 清空/重置数据 → 获取新数据 → loading = false
```
这会导致页面先显示加载状态，然后显示空状态或数据，造成视觉闪烁。

### 2. 保存操作后的全量刷新
在管理页面中，保存/删除操作后调用 `fetchData()` 重新获取全部数据，导致：
- 整个列表重新渲染
- loading 状态切换导致闪烁

### 3. 轮询刷新时的数据替换
`DashboardView.vue` 中的订单轮询虽然避免了 loading 状态，但数据替换仍可能导致列表重渲染闪烁。

### 4. 主题异步加载闪烁
`app.ts` 中主题从 IndexedDB 异步加载，页面初始渲染时可能出现主题闪烁。

---

## 解决方案

### 策略一：乐观更新（Optimistic Update）
在保存/删除操作时，先更新本地数据，API 成功后保持，失败则回滚。

**涉及文件：**
- `src/admin/views/DishesView.vue`
- `src/admin/views/TablesView.vue`
- `src/admin/views/DashboardView.vue`

### 策略二：数据保留加载
加载数据时保留旧数据直到新数据加载完成，避免中间的空状态。

**涉及文件：**
- `src/client/views/HomeView.vue`
- `src/client/views/OrdersView.vue`
- `src/admin/views/DishesView.vue`
- `src/admin/views/TablesView.vue`
- `src/admin/views/DashboardView.vue`

### 策略三：平滑过渡动画
为数据加载添加淡入淡出过渡效果，使用 Vue 的 `<Transition>` 组件。

**涉及文件：**
- 各视图组件

### 策略四：主题预加载
在 HTML 中预先设置默认主题，避免异步加载时的闪烁。

**涉及文件：**
- `index.html`
- `src/stores/app.ts`

---

## 具体修改计划

### 1. DishesView.vue - 菜品管理页面

**问题：** 保存/删除菜品后调用 `fetchData()` 导致列表闪烁

**修改方案：**
- 保存菜品时使用乐观更新：直接修改本地 `dishes` 数组
- 删除菜品时直接从数组中移除，无需重新获取
- 添加菜品时将新菜品追加到数组开头
- 仅在首次加载和需要完整刷新时调用 `fetchData()`

### 2. TablesView.vue - 桌位管理页面

**问题：** 同 DishesView，保存/删除后全量刷新

**修改方案：**
- 保存桌位时乐观更新本地 `tables` 数组
- 删除桌位时直接从数组中移除
- 状态变更时直接更新对应项

### 3. DashboardView.vue - 订单管理页面

**问题：** 
- 首次加载时统计数据和订单列表闪烁
- 轮询刷新时订单列表可能闪烁

**修改方案：**
- 统计卡片：保留旧数据直到新数据加载完成
- 订单列表：使用差异更新，比较新旧数据后只更新变化的项
- 轮询刷新：保持当前滚动位置，平滑更新数据

### 4. HomeView.vue - 客户端首页

**问题：** 首次加载时菜品列表闪烁

**修改方案：**
- 保留旧数据直到新数据加载完成
- 添加骨架屏过渡动画
- 使用 `<Transition>` 组件实现平滑切换

### 5. OrdersView.vue - 客户端订单列表

**问题：** 加载时显示 loading 状态导致闪烁

**修改方案：**
- 保留旧订单数据直到新数据加载完成
- 添加骨架屏过渡

### 6. app.ts - 主题加载

**问题：** 主题异步加载导致页面闪烁

**修改方案：**
- 在 `index.html` 中预先读取 localStorage 设置主题
- 使用 CSS 过渡平滑切换主题

### 7. index.html - 添加主题预加载脚本

**修改方案：**
- 添加内联脚本在页面加载前设置主题
- 避免主题闪烁

# 调试工具折叠菜单改造计划

## 摘要

将侧边栏中的「调试工具」入口从单一按钮改造成可折叠的父菜单，下面挂两个子菜单项：「SQL 查询」与「API 调试」。点击子项分别跳转到独立子路由 `/admin/debug/sql` 与 `/admin/debug/api`；当路由位于 `/admin/debug` 下时，该折叠菜单默认展开。原 `DebugToolsPanel.vue` 顶部的 SQL/API Tab 按钮将被移除，由路由驱动当前展示的面板。

## 改造方案

### 1. 路由：新增 SQL/API 子路由

**文件**：[src/router/index.ts](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/router/index.ts)

- 在 `adminRoutes` 中保留 `/admin/debug` 路由，并新增两个同级路由：
  - `/admin/debug/sql` -> `DebugView.vue`，`meta.title = 'SQL 查询'`
  - `/admin/debug/api` -> `DebugView.vue`，`meta.title = 'API 调试'`
- 保持 `/admin/debug` 仍可访问（向后兼容，默认展示 SQL 面板）。
- 这样无需拆分 `DebugView` 或 `DebugToolsPanel`，改动最小。

### 2. 侧边栏：实现「调试工具」折叠菜单

**文件**：[src/admin/views/LayoutView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/admin/views/LayoutView.vue)

- 引入 `Terminal`、`Send`、`ChevronRight` 图标（来自 `lucide-vue-next`）。
- 将 `navItems` 中的调试项改造为支持 `children` 的结构：

```ts
{
  icon: Code,
  label: '调试工具',
  path: '/admin/debug',
  children: [
    { icon: Terminal, label: 'SQL 查询', path: '/admin/debug/sql' },
    { icon: Send, label: 'API 调试', path: '/admin/debug/api' }
  ]
}
```

- 新增响应式状态 `debugMenuExpanded`，默认 `false`。
- 监听 `route.path`，当路径以 `/admin/debug` 开头时自动将 `debugMenuExpanded` 设为 `true`。
- 在模板中渲染折叠菜单：
  - 父按钮：左侧图标 + 标签 + 右侧 `ChevronRight`，点击切换展开/收起。
  - 子菜单容器：仅在展开且侧边栏未折叠时显示，内部循环渲染两个子项按钮。
  - 子项使用缩进的 `nav-item` 样式，激活时沿用 `nav-item-active`。
- 折叠动画：使用 `grid-template-rows` 或 `max-height` + opacity 实现平滑展开/收起。
- 侧边栏折叠状态：当 `sidebarCollapsed` 为 `true` 时，仅显示父级图标按钮；点击图标直接跳转 `/admin/debug/sql`（收起状态下不展示子菜单，保持简洁）。

### 3. 调试面板：移除 Tab 按钮，改为路由驱动

**文件**：[src/admin/components/DebugToolsPanel.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/admin/components/DebugToolsPanel.vue)

- 引入 `useRoute`。
- 将 `activeTab` 从 `ref<'sql' | 'api'>('sql')` 改为 `computed`：

```ts
const route = useRoute()
const activeTab = computed<'sql' | 'api'>(() => {
  return route.path === '/admin/debug/api' ? 'api' : 'sql'
})
```

- 删除模板顶部的 `.debug-tabs` 区域（两个 `debug-tab` button）。
- 保留 `.tab-content` 与 SQL/API 面板的其余所有逻辑与样式。
- 删除 `debug-tabs`、`.debug-tab`、`.tab-content` 中不再需要的样式（保留 `.tab-content` 的 `max-height` 与 `overflow-y`）。

### 4. 调试页面：可选微调

**文件**：[src/admin/views/DebugView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/admin/views/DebugView.vue)

- 页面标题保持「调试工具」即可，无需按子路由变化；如需变化，可通过 `useRoute` 读取 `meta.title` 动态展示。
- 本次计划保持标题不变，以减小影响面。

## 假设与决策

1. **路由选择**：采用同级路由 `/admin/debug/sql` 与 `/admin/debug/api`，而非 Vue Router 嵌套路由。原因是现有 `DebugView.vue` + `DebugToolsPanel.vue` 结构无需拆分成布局组件，改动最小，且能满足侧边栏子菜单跳转需求。
2. **折叠动画**：侧边栏展开时使用平滑展开动画；侧边栏折叠时仅保留父级图标入口，点击后跳转默认 SQL 页面，不再展开子菜单（避免在小宽度侧栏内排版混乱）。
3. **默认展开**：只要当前路由在 `/admin/debug` 前缀下，调试工具菜单就默认展开，便于用户感知当前所在子项。
4. **激活状态**：父菜单在任一子路由激活时保持高亮；子菜单项在自身路由激活时高亮。
5. **移除面板内 Tab**：因为侧边栏子菜单已经承担切换职责，面板顶部 Tab 按钮将完全移除，避免功能重复。

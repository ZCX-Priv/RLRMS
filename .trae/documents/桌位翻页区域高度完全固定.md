# 桌位翻页区域高度完全固定

## 概述

修复桌位翻页时区域高度仍轻微变化的问题。上一次修复使用了 `minmax(72px, auto)`，但 `auto` 导致有内容的行比空行高约 6px，翻页时仍能察觉到高度差异。

## 当前状态分析

**文件**：[OrderConfirmView.vue](file:///c:/Users/赵晨旭/Desktop/红灯笼食府管理系统/src/client/views/OrderConfirmView.vue#L655)

当前样式（第 655 行）：
```css
grid-template-rows: repeat(3, minmax(72px, auto));
```

**问题原因**：
- `minmax(72px, auto)` 中，`auto` 会根据 grid item 内容计算实际行高
- 有桌位的行：padding(16px×2=32px) + 内容(table-name 14px + gap 4px + table-capacity 12px ≈ 46px) ≈ **78px**
- 空行（无 grid item）：严格取最小值 **72px**
- 每行差异约 6px，3 行累计约 6-12px，用户能察觉到"稍微变小"

## 提议变更

**文件**：`src/client/views/OrderConfirmView.vue`

**修改 `.table-grid` 的 `grid-template-rows`**（第 655 行）：

```css
/* 修改前 */
grid-template-rows: repeat(3, minmax(72px, auto));

/* 修改后 */
grid-template-rows: repeat(3, 80px);
```

**原理**：
- 将 `minmax(72px, auto)` 改为固定值 `80px`
- 无论行内有无桌位，每行高度严格为 80px
- 80px 足够容纳桌位卡片内容（实际需要约 78px），留 2px 余量
- 翻页时 3 行总高度始终为 240px + gap，完全一致

**为什么选 80px**：
- 桌位卡片实际内容高度约 78px（padding 32px + 内容 46px）
- 80px 留 2px 余量，确保内容不被压缩
- 桌位名称通常很短（"小桌01"、"包厢02"等），不会换行导致溢出

## 假设与决策

1. **固定行高 80px**：桌位卡片内容简单（名称 + 容量），80px 足够。若未来桌位名称很长导致换行，可再调整。

2. **不使用 minmax**：`minmax` 的 `auto` 是导致高度不一致的根因，改用固定值才能彻底解决。
